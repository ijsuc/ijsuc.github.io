React 路由匹配

2016/10/17

一个路径有确定它是否“匹配”的URL三个属性:

  1. 嵌套
  2. 它的 path
  3. 它的优先级

嵌套

反应路由器使用嵌套路线的概念，让你申报嵌套组调用指定的URL时应该呈现的观点。嵌套路线布置成树状结构。为了找到一个匹配，路由器做出反应穿越路线配置深度优先搜索该网址相符的路线。

路径语法

路线路径是一个字符串模式，用于匹配的URL（或其中的一部分）。路由路径字面解释，除以下特殊符号:

  • :paramName -匹配的URL段到下一个/，?或#。匹配的字符串被称为参数
  • () - 包装的URL的一部分是可选
  • * - 匹配所有字符（非贪婪）到在模式的下一个字符，或者如果没有，并创建一个URL的结束参数splat
  • **-匹配，直到下一次所有字符（贪婪）/，?或#并创建一个参数splat

如果路由使用相对path，它建立在积累的path祖先。嵌套的路由可能是由退出的这种行为用一个绝对的path。
